คู่มือผู้พัฒนา - Trash For Coin

เอกสารสำหรับผู้พัฒนาระบบ Trash For Coin

เวอร์ชัน 1.0 | Flask + MySQL + Bootstrap 5

สถาปัตยกรรมระบบ

Backend (Flask)

  • Framework: Flask (Python)
  • Database: MySQL
  • ORM: mysql-connector-python
  • Authentication: Flask Sessions
  • PDF Generation: xhtml2pdf

Frontend

  • Templates: Jinja2
  • CSS Framework: Bootstrap 5.3.0
  • Icons: Bootstrap Icons
  • Fonts: Google Fonts (Inter, Sarabun)
  • JavaScript: Vanilla JS

3. ตั้งค่าฐานข้อมูล

CREATE DATABASE project_bin;
USE project_bin;

-- สร้างตาราง users
CREATE TABLE tbl_users (
    id INT AUTO_INCREMENT PRIMARY KEY,
    firstname VARCHAR(100) NOT NULL,
    lastname VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role VARCHAR(50) DEFAULT 'member',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- สร้างตาราง category
CREATE TABLE tbl_category (
    id INT AUTO_INCREMENT PRIMARY KEY,
    category_id VARCHAR(50) UNIQUE NOT NULL,
    category_name VARCHAR(100) NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- สร้างตาราง products
CREATE TABLE tbl_products (
    id INT AUTO_INCREMENT PRIMARY KEY,
    products_id VARCHAR(50) UNIQUE NOT NULL,
    products_name VARCHAR(200) NOT NULL,
    stock INT DEFAULT 0,
    price DECIMAL(10,2) DEFAULT 0.00,
    category_id VARCHAR(50),
    description TEXT,
    barcode_id VARCHAR(100),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (category_id) REFERENCES tbl_category(category_id)
);

-- สร้างตาราง order
CREATE TABLE tbl_order (
    id INT AUTO_INCREMENT PRIMARY KEY,
    order_id VARCHAR(50) NOT NULL,
    products_id VARCHAR(50),
    products_name VARCHAR(200),
    barcode_id VARCHAR(100),
    quantity INT NOT NULL,
    disquantity INT DEFAULT 0,
    email VARCHAR(100),
    order_date TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (products_id) REFERENCES tbl_products(products_id)
);

บทบาทผู้ใช้งาน (User Roles)

Role Description Permissions
root_admin ผู้ดูแลระบบสูงสุด จัดการผู้ใช้งานทั้งหมด, เข้าถึงข้อมูลทั้งหมด
administrator ผู้ดูแลระบบ จัดการผู้ใช้งาน (ยกเว้น root_admin), จัดการข้อมูล
moderator ผู้ดูแลเนื้อหา จัดการสินค้า, หมวดหมู่, คำสั่งซื้อ
member สมาชิก ดูและจัดการคำสั่งซื้อของตนเอง
viewer ผู้ดูข้อมูล ดูข้อมูลเท่านั้น

ระบบยืนยันตัวตนและการควบคุมสิทธิ์

Role-Based Access Control (RBAC)

ระบบใช้ Decorator pattern สำหรับการควบคุมสิทธิ์การเข้าถึง:

@role_required(['root_admin', 'administrator', 'moderator'])
def tbl_category():
    # เฉพาะ root_admin, administrator, moderator เท่านั้น
    pass

@role_required(['root_admin', 'administrator', 'moderator', 'member', 'viewer'])
def tbl_order():
    # ผู้ใช้งานทุกประเภท (ยกเว้น guest)
    pass

Session Management

ระบบใช้ Flask Sessions สำหรับการจัดเก็บข้อมูลการล็อกอิน:

# การล็อกอิน
session['loggedin'] = True
session['id'] = account['id']
session['email'] = account['email']
session['firstname'] = account['firstname']
session['lastname'] = account['lastname']
session['role'] = account['role']

# การล็อกเอาท์
session.pop('loggedin', None)
session.pop('id', None)
session.pop('email', None)
session.pop('firstname', None)
session.pop('lastname', None)
session.pop('role', None)

API Documentation

API Endpoints

Method Endpoint Description Required Role
GET /api/products ดึงข้อมูลสินค้าทั้งหมด All logged-in users
GET /api/categories ดึงข้อมูลหมวดหมู่ทั้งหมด All logged-in users
GET /api/users ดึงข้อมูลผู้ใช้งาน root_admin, administrator
GET /export_products_csv ส่งออกข้อมูลสินค้าเป็น CSV root_admin, administrator, moderator
GET /export_orders_pdf ส่งออกรายงานคำสั่งซื้อเป็น PDF root_admin, administrator, moderator, member

ตัวอย่างการใช้งาน API

// JavaScript - ดึงข้อมูลสินค้า
fetch('/api/products')
    .then(response => response.json())
    .then(data => {
        console.log(data);
    });

// Python - เรียกใช้ API
import requests

response = requests.get('http://localhost:5000/api/products')
if response.status_code == 200:
    products = response.json()
    print(products)

ความปลอดภัย

มาตรการความปลอดภัย

  • Session Management: ใช้ Flask Sessions สำหรับการจัดเก็บข้อมูลการล็อกอิน
  • Role-Based Access: ควบคุมการเข้าถึงตามบทบาท
  • Input Validation: ตรวจสอบข้อมูลที่รับเข้ามา
  • SQL Injection Prevention: ใช้ Parameterized Queries

ข้อแนะนำด้านความปลอดภัย

สำคัญ: ในการใช้งานจริง ควรจะ:
  • ใช้ HTTPS สำหรับการเชื่อมต่อทั้งหมด
  • เข้ารหัสรหัสผ่านด้วย bcrypt หรือ argon2
  • ใช้ strong secret key สำหรับ Flask sessions
  • ตั้งค่า rate limiting เพื่อป้องกัน brute force attacks
  • ใช้ environment variables สำหรับ sensitive data

ตัวอย่างการเข้ารหัสรหัสผ่าน

from werkzeug.security import generate_password_hash, check_password_hash

# การเข้ารหัสรหัสผ่าน
hashed_password = generate_password_hash('user_password')

# การตรวจสอบรหัสผ่าน
if check_password_hash(hashed_password, 'user_password'):
    print("รหัสผ่านถูกต้อง")

การ Deploy และการผลิต

1. การเตรียมสำหรับ Production

# ปิด debug mode
app.run(debug=False)

# หรือใช้ environment variable
import os
debug_mode = os.environ.get('FLASK_DEBUG', 'False').lower() == 'true'
app.run(debug=debug_mode)

2. การใช้งานกับ WSGI Server

# ติดตั้ง Gunicorn
pip install gunicorn

# รันด้วย Gunicorn
gunicorn --bind 0.0.0.0:8000 app:app

3. ตั้งค่า Environment Variables

# .env file
FLASK_SECRET_KEY=your-super-secret-key-here
MYSQL_HOST=localhost
MYSQL_USER=root
MYSQL_PASSWORD=your-password
MYSQL_DATABASE=project_bin
FLASK_DEBUG=False

4. การตั้งค่า Web Server (Nginx)

server {
    listen 80;
    server_name your-domain.com;
    
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
    
    location /static {
        alias /path/to/your/app/static;
    }
}

การแก้ปัญหา

ปัญหาที่พบบ่อย

สาเหตุ: การตั้งค่าฐานข้อมูลไม่ถูกต้อง
วิธีแก้:
  1. ตรวจสอบว่า MySQL เริ่มต้นแล้ว
  2. ตรวจสอบ username, password, database name
  3. ตรวจสอบว่าผู้ใช้มีสิทธิ์เข้าถึงฐานข้อมูล

สาเหตุ: ไม่มีผู้ใช้งานในระบบ
วิธีแก้:
  1. สร้างผู้ใช้งานผ่านหน้าสมัครสมาชิก
  2. หรือเพิ่มผู้ใช้งานโดยตรงในฐานข้อมูล
  3. ตรวจสอบว่าตาราง tbl_users มีข้อมูลอยู่

สาเหตุ: ไม่มี xhtml2pdf library
วิธีแก้:
  1. ติดตั้ง xhtml2pdf: pip install xhtml2pdf
  2. ตรวจสอบว่า template PDF อยู่ที่ถูกต้อง
  3. ตรวจสอบ permission ในการเขียนไฟล์
